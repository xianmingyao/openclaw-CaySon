## 核心定位

**一句话总结**：让 AI Scientist 只需交付 algorithm.py + yaml，Alchemy 负责把科研基础设施连接起来。

**Slogan**：让自动化 AI 科研只剩最后一块拼图

## 解决问题

当前自动化 AI 科研系统的**核心矛盾**：
- 科学发现与科研基础设施耦合在同一条链路里
- 工程负担重，拖累了科研效率
- 难以规模化开展实验

## 核心架构

### 两层分离设计

```
┌─────────────────────────────────────┐
│  AI Scientist（算法开发层）          │  ← 用户只需关注这里
├─────────────────────────────────────┤
│  Alchemy 炼丹炉（研究环境执行层）  │  ← 复杂工程自动化处理
└─────────────────────────────────────┘
```

### 工作流程

1. **用户提供 Seed Baseline**（种子基线）
   - 可以是人类研究者的前沿算法（如 AAAI 2025 的工作）
   
2. **AI Scientist 自动演化**
   - 提出假设
   - 生成方法
   - 优化算法

3. **Alchemy 自动化处理**
   - 实验环境搭建
   - 超参数配置
   - 结果收集与分析
   - 迭代优化循环

## 核心功能

| 功能 | 说明 |
|------|------|
| algorithm.py + yaml 交付模式 | 用户只需提供核心算法代码和配置文件 |
| 标准化接口 | 将复杂异构的 AI 工程统一接口 |
| 多模态推荐任务 | AI Scientist 可在无人类干预下优化算法 |
| Seed Baseline 机制 | 作为科学假设生成的起点 |
