## CHAPTER 01 - Skills核心机制

### YAML头部技术要点

| 要点 | 说明 |
|------|------|
| **Name唯一性** | 必须全小写，且与文件夹名称1:1严格对齐 |
| **Description意图识别** | 它是LLM进行意图路由的唯一上下文 |

### 描述编写规范

**弱描述示例**：
```yaml
description: "处理财务报表"
```
- 问题：缺乏触发场景描述、意图模糊、命中准确率低
- 命中率：~45%

**强描述示例**：
```yaml
description: "当用户上传PDF财务报表时，自动解析并提取关键财务指标"
```
- 结构：动作(What) + 触发时机(When)
- 命中率：~92%

## CHAPTER 02 - Markdown Body 思维链艺术

### CoT三要素

| 要素 | 说明 |
|------|------|
| **1. 判定条件** | 在正文中二次强化场景 |
| **2. 分步指南** | 将复杂任务拆解为原子化操作 |
| **3. 异常分支** | 为Agent预设容错路径 |

### CoT流程图示例

```
1. 接收用户输入
   → 解析PDF文件路径与提取需求
2. 环境检查
   → 验证系统环境（Windows/Linux/Mac）
3. 执行提取
   → 调用pdfplumber库提取文本内容
4. 异常处理
   → 超时重试（最多3次）→ 失败返回错误信息
```
