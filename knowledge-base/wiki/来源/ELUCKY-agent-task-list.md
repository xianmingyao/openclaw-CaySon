```
TK-Anchor（挂靠）Agent 任务流

📋 挂靠前准备

└── A3: 关系映射
    ├── A3.1: 按账号类型分配根基账号
```

## 六、Agent Task执行监控看板

### 6.1 任务状态定义

| 状态 | 说明 | 触发条件 |
|------|------|---------|
| PENDING | 待执行 | 任务创建 |
| RUNNING | 执行中 | 任务开始 |
| SUCCESS | 成功 | 任务完成 |
| FAILED | 失败 | 任务失败 |
| RETRYING | 重试中 | 失败自动重试 |
| BLOCKED | 阻塞 | 依赖未完成 |

### 6.2 核心监控指标

| 指标 | 计算方式 | 健康阈值 |
|------|---------|---------|
| Task成功率 | 成功数/总数 | >95% |
| Task平均耗时 | 总耗时/任务数 | <预期150% |
| 异常账号率 | 异常数/账号总数 | <5% |
| 挂靠覆盖率 | 已挂靠/应挂靠 | >90% |
