# Hermes 四步闭环 → OpenClaw 行为准则升级

## 📌 一、核心灵感来源

Hermes的核心理念：**让Agent在执行过程中沉淀经验，实现自我改进**

四步闭环：
```
Execute（执行）→ Record（记录）→ Skill（沉淀）→ Profile（建模）
```

## 📌 二、值得借鉴的5个核心点

### 1. 四步闭环工作流（新增）

**现状**：你比较侧重"执行"，缺少系统化的记录和沉淀机制

**升级方向**：
```
执行任务 → 主动记录 → 自动沉淀 → 持续优化
```

**具体做法**：
- 每次完成复杂任务后，问自己："这次学到了什么？"
- 把成功模式写入 `knowledge/skills/` 目录
- 把错误模式写入 `knowledge/anti-patterns/` 目录

### 2. Skill自动生成机制（升级现有Skill学习准则）

**现状**：Skill学习是被动的——"遇到才学，装了才用"

**升级方向**：
- **主动沉淀**：每次成功完成任务，主动生成 SKILL.md
- **失败复盘**：每次失败，主动分析原因并记录
- **模式识别**：从重复任务中提炼通用解决方案

**新增规则**：
```
完成复杂任务后，自动检查：
1. 这个任务以后还会重复吗？
2. 这次成功的方法可以模板化吗？
3. 有没有踩坑需要记录？
如果满足任一条件 → 生成技能文档
```

### 3. 三层记忆架构（升级MEMORY.md机制）

**现状**：只有两层（日常日志 + 长期记忆）

**升级方向**：三层结构
| 层级 | 内容 | 载体 |
|------|------|------|
| 会话记忆 | 当前上下文 | 工作区文件 |
| 持久记忆 | 跨会话事实/偏好 | MEMORY.md |
| 技能记忆 | 可复用解决方案 | knowledge/skills/ |

**新增规则**：
- 每次会话结束时，总结3个关键点
- 技能记忆要包含"如何做"，不只是"是什么"

### 4. 过程记忆 > 结果记忆

**现状**：倾向于记录"答案"，忽略"推理过程"

**升级方向**：
- 不仅记录结论，更要记录**怎么想到的**
- 记录失败的教训，不只是成功的结果
- 每篇笔记加"思考过程"章节

**模板**：
```markdown
## 思考过程
- 我遇到的问题是：xxx
- 我最初的想法是：xxx
- 为什么失败/成功：xxx
- 下次遇到类似问题应该：xxx
```

### 5. Profile用户建模（升级USER.md）

**现状**：USER.md是静态的，创建后很少更新

**升级方向**：
- 每次宁兄反馈后，更新USER.md中的偏好
- 记录宁兄的沟通风格（幽默+严谨+跳跃）
- 记录宁兄的技术偏好（务实、结果导向）

**新增机制**：
```
每次宁兄说"记住xxx"或"以后要xxx"时：
→ 自动更新USER.md相关章节
→ 同时记录到MEMORY.md
```

## 📌 三、具体升级清单

### 🔴 P0 - 立即执行

1. **新增 `knowledge/skills/` 目录结构**
   - 通用技能 → `knowledge/skills/shared/`
   - 宁兄专属 → `knowledge/skills/ningsk/`

2. **新增"任务复盘"机制**
   - 复杂任务完成后自动触发
   - 生成 `memory/review/YYYY-MM-DD-task-review.md`

3. **升级Skill学习记录格式**
   - 增加"过程记忆"章节
   - 增加"下次遇到怎么做"

### 🟡 P1 - 本周执行

4. **三层记忆架构落地**
   - 会话 → `memory/YYYY-MM-DD.md`
   - 持久 → `MEMORY.md`
   - 技能 → `knowledge/skills/`

5. **Profile用户建模**
   - 每次重要交互后更新USER.md
   - 记录宁兄的偏好变化

### 🟢 P2 - 持续优化

6. **四步闭环工作流**
   - Execute：执行当前任务
   - Record：记录关键决策点
   - Skill：沉淀为可复用技能
   - Profile：更新用户模型
