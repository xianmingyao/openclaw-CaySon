## 1. 🎯 这是什么

### 核心命题
**Agent 的持续学习可以发生在三个层面**，每个层面有不同的成本、速度和效果。

```

│  │ 模型权重    │  │ 执行框架    │  │ 运行时配置  │   │
│  └─────────────┘  └─────────────┘  └─────────────┘   │

```

## 2. 📊 三层详细对比

| 层级 | 学习内容 | 更新方式 | 成本 | 速度 | 粒度 | 上限 |
|------|---------|---------|------|------|------|------|
| **Model 层** | 模型权重 | 微调/强化学习 | 🔴 最高 | 🐢 最慢 | - | ⭐⭐⭐⭐⭐ |
| **Harness 层** | 执行方式 | 代码更新 | 🟡 中等 | 🟡 中等 | ⭐⭐⭐⭐ | ⭐⭐⭐ |

## 3. 🔍 各层详解

### 1️⃣ Model 层（模型层）

**类比：** 🚗 **引擎/大脑**

| 维度 | 说明 |
|------|------|
| **组成** | 神经网络参数 |
| **决定因素** | Agent 能力的**上限** |
| **更新方式** | 微调（fine-tuning）或强化学习（RL） |

**问题：**
- ❌ **灾难性遗忘**：学新忘旧
- ❌ 难以按用户/租户分别更新
- ❌ 训练成本极高，周期最长

**结论：** 影响上限最高，但问题最多

---

### 2️⃣ Harness 层（执行框架层）

**类比：** 🏎️ **底盘/车架**

| 维度 | 说明 |
|------|------|
| **组成** | Prompt 逻辑、工具调用逻辑、工作流编排 |
| **决定因素** | Agent 的**基本运作模式** |
| **更新方式** | 代码更新 |

**特点：**
- ✅ 学习机制更像**工程闭环**
- ✅ 更新速度**最快**
- ✅ 粒度**最细**

**类比理解：** 就像一个优秀的工程师不断优化使用工具的方式

---

### 3️⃣ Context 层（运行时层）

**类比：** 📍 **GPS 导航/赛前路书**

| 维度 | 说明 |
|------|------|
| **组成** | 指令/技能（instructions/skill）、工具/记忆（tools/memory） |
| **决定因素** | Agent **当前**的行为 |
| **更新方式** | **热更新**（运行时可更改） |

**特点：**
- ✅ 方式**最灵活**
- ✅ 可按**用户、组织、团队、租户**分别维护
- ✅ **见效最快**

**类比：** 赛前路书，每一圈比赛都可以根据情况进行修改

---

## 4. ⚡ 三层对比总结

### 神级类比

```
Model 层      → 🚗 引擎       → 决定你能跑多快（智商）
Harness 层   → 🏎️ 底盘      → 决定你能跑什么赛道（思维模式）
Context 层   → 📍 GPS导航   → 决定你当前往哪走（眼前地图）
```

### 通俗理解

| 层级 | 通俗解释 | 可改变程度 |
|------|---------|-----------|
| Model | 你的**智商** | 天生的，改不了太快 |
| Harness | 你的**思维方式** | 可以训练，但慢 |
| Context | 你眼前的**地图和路况** | 随时可以更新，最快 |

## 5. 💡 关键洞察

### Trace 是三层学习的共同原料

```
Trace（执行轨迹）
    ↓
┌─────────────┐
│ Model 层    │ → 从 Trace 学习模式，更新权重
├─────────────┤
│ Harness 层  │ → 从 Trace 优化执行方式
├─────────────┤
│ Context 层  │ → 从 Trace 调整运行时配置
└─────────────┘
```

### 可观测性是基础设施

> 企业应考虑**权限边界**，选择合适的层面进行学习。

- Model 层需要考虑数据安全和隐私
- Harness 层需要工程化能力
- Context 层最灵活，适合多租户场景

---

## 6. 🎯 优化优先级建议

| 场景 | 优先级 | 原因 |
|------|--------|------|
| **快速见效** | Context 层 | 加个 Skill 就变强 |
| **系统稳定** | Harness 层 | 优化执行框架 |
| **突破上限** | Model 层 | 换更强的模型 |

### 为什么 Context 层最实用？

```
Context 层 = 运行时配置 = Skills + Memory
                ↓
         "热更新"能力
         不改模型、不改代码
         直接加个 Skill 就变强
```

**这就是 OpenClaw 的 Skills 为什么重要！**
