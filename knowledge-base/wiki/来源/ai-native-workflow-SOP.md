## 🔴 关键检查点（宁兄必须遵守）

### ❌ 常见错误

| 错误 | 正确做法 |
|------|---------|
| 直接写代码，没有先做调研 | **必须** 先 Google/GitHub/知网 搜索 |
| 没有设计稿直接开发 | **必须** 先 Figma 出设计稿 |
| 不用 AI IDE (Cursor/Trae) | **必须** 用 AI IDE 开发 |
| 不建立知识库直接做 | **必须** NotebookLM/Obsidian 整理 |
| 不用 MCP 协议连接 | **建议** MCP 接入外部工具 |
| 代码直接上线 | **必须** GitHub + Vercel 流程 |

### ✅ 每个阶段的产出物

| 阶段 | 产出物 | 检查点 |
|------|--------|-------|
| 1. 知识获取 | CodeWiki 文档 | 有 Google/GitHub/知网 参考 |
| 2. 想法原型 | 产品方向文档 | 有明确的 Idea |
| 3. UI设计 | Figma 设计稿 | 有完整设计图 |
| 4. 知识库 | NotebookLM/Obsidian 库 | 有 RAG 知识 |
| 5. AI逻辑 | Gemini Prompt 设计 | 有 Skills 定义 |
| 6. 开发 | AI IDE 项目 | 有 Cursor/Trae 项目 |
| 7. 部署 | 上线产品 | 有 GitHub + Vercel 流程 |

## 🐂 CaySon 职责

**当宁兄流程出错时，CaySon 必须：**

1. **立即指出** 哪个阶段出了问题
2. **说明错误原因** 为什么这样做不对
3. **给出正确做法** 具体应该怎么改
4. **提供示例** 正确的代码/文档/流程示例

---

## 📝 流程执行检查清单

**每个项目启动时，宁兄需要确认：**

```
□ 阶段1：是否做了 Google/GitHub/知网 调研？
□ 阶段2：是否有明确的产品 Idea？
□ 阶段3：是否有 Figma 设计稿？
□ 阶段4：是否建立了知识库（NotebookLM/Obsidian）？
□ 阶段5：是否设计了 Gemini Skills？
□ 阶段6：是否使用 Cursor/Trae 开发？
□ 阶段7：是否通过 GitHub + Vercel 部署？
□ 阶段7：最终是否上线？
```

**如果任何一个阶段缺失，CaySon 将立即提醒并要求补全。**

---

## 🚨 流程异常处理

| 异常情况 | CaySon 响应 |
|---------|------------|
| 跳过调研直接开发 | 🔴 立即暂停，要求补全调研 |
| 没有设计稿开始写代码 | 🔴 立即暂停，要求先出设计稿 |
| 不用 AI IDE | 🟡 建议使用，但不强制 |
| 没有知识库 | 🟡 建议建立，但不强制 |
| 上线前没走 GitHub | 🔴 立即暂停，必须走流程 |
