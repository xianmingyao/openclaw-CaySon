## 📋 技术文档标准格式（铁律）

> 所有技术相关内容必须按此格式书写！

### 必须包含的章节：

```
1. 🎯 这是什么（简介）
2. 📝 关键功能点
3. ⚡ 怎么使用
4. ✅ 优点
5. ❌ 缺点
6. 🎬 使用场景
7. 🔧 运行依赖环境
8. 🚀 部署使用注意点
9. 🕳️ 避坑指南
10. 📊 总结
```

---

## 🎨 格式要素

### 1. Emoji 图标系统
```
✅ 正面反馈（好消息、成功状态）
❌ 负面反馈（问题、错误）
🔴 严重问题/警告
🎯 核心要点
📝 操作步骤
🔧 环境配置
🕳️ 避坑指南
🚀 部署注意点
💡 提示说明
⭐ 评分/推荐
```

### 2. 标题格式
```
✅ 一级强调：Emoji + 加粗文字（如"✅ 优点"）
❌ 二级强调：Emoji + 加粗文字（如"❌ 缺点"）
📝 步骤标题：加粗 + 数字（如"第一步：xxx"）
```

### 3. 代码块格式（必须严格遵守）
```
┌─ bash ──────────────────────────────────────┐
│ rd /s /q "C:\Users\...\feishu"             │
└─────────────────────────────────────────────┘

┌─ yaml ──────────────────────────────────────┐
│ name: Claude Code Review                     │
│ on: [pull_request]                          │
└─────────────────────────────────────────────┘
```
- 外框：浅灰色圆角边框
- 背景：极浅灰色
- 顶部栏：标注语言类型（bash/yaml/python/json）
- 字体：等宽字体(Monospace)
- 语法高亮：命令深色，路径字符串绿色

### 4. 列表格式
```
有序列表（数字）：用于具体步骤
• 无序列表（实心圆点）：用于列举要点
• 每行简洁，方便快速扫描
```

### 5. 表格格式（可选）
```
| 项目 | 说明 |
|------|------|
| Star | 6.5k |
| Fork | 1.6k |
```
