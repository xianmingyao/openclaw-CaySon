## 1. 🎯 这是什么

### 核心理念

**用 LLM 把原始资料（论文/文章/代码）编译成 wiki 知识库**，在 Obsidian 中查看和查询。

- 100 篇文章 / 40 万字规模下 **不需要 RAG**

- LLM 自维护索引就够用

- 查询结果反哺 wiki，知识越滚越大

### 工作流三步曲

┌─────────────────────────────────────────────────────────┐

│  STEP 01 收集                                            │

│  raw/ 目录 ← 论文/文章/GitHub/数据集/截图                 │

│           ← Obsidian Web Clipper (网页 → .md)            │

└─────────────────────────────────────────────────────────┘

↓

┌─────────────────────────────────────────────────────────┐

│  STEP 02 编译（LLM）                                     │

│  raw/ → wiki/ 知识库                                     │

│  - 提取概念                                              │

│  - 写文章                                                │

│  - 建分类目录                                            │

│  - 加反向链接 (bidirectional links)                       │

│  输出：Obsidian 中查看的 wiki 知识库                      │

└─────────────────────────────────────────────────────────┘

↓

┌─────────────────────────────────────────────────────────┐

│  STEP 03 查询                                            │

│  直接问 LLM → LLM 在 wiki 中查找答案                     │

│  - 100篇/40万字以下 → 无需 RAG，LLM 自维护索引            │

│  - 查询结果反哺 wiki → 知识越滚越大                      │

└─────────────────────────────────────────────────────────┘

## 3. ⚡ 编译阶段详解

### LLM 编译任务

让 LLM 将 raw/ 目录编译成 wiki，包括：

| 任务 | 说明 |

|------|------|

| **提取概念** | 从原始资料中识别和提取关键概念 |

| **写文章** | 将相关概念组织成完整的文章 |

| **建分类目录** | 建立知识分类体系 |

| **加反向链接** | 建立双向链接，形成知识网络 |

### 编译 Prompt 示例（推测）

你是一个知识整理专家。请将 raw/ 目录下的所有资料编译成一个 wiki 知识库。

要求：

1. 提取每个文件的核心概念

2. 为每个概念写一篇简短的解释文章

3. 建立分类目录结构

4. 添加双向链接（bidirectional links）关联相关概念

5. 输出格式：Markdown

请开始编译。

## 4. ✅ 查询阶段详解

### 为什么不需要 RAG？

> Karpathy 的观点：100 篇/40 万字规模下，LLM 的上下文窗口足够大，**直接把所有 wiki 内容放进 context 就够了**。

传统 RAG：检索 → 生成（外部知识增强）

↓

Karpathy：编译 → 自维护索引 → 查询（内化知识直接答）

### 知识库的自我完善

查询 → 找到答案 → 好的结果归档回 wiki

↓

wiki 越来越完善

↓

后续查询越来越准

LLM 会定期检查数据，好的查询结果会被归档回 wiki。

## 5. 🔥 核心洞察

### Karpathy 的长期愿景

> Karpathy 考虑**将知识写进模型权重**，认为这是一个巨大的产品机会。

这意味着：

- 未来的 AI 不仅能查询知识

- 还能把新知识**内化**到模型权重中

- 知识管理和模型训练融为一体
