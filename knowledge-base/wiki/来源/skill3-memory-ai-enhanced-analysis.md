## 🎯 这是什么（核心概念）

### 双重记忆系统 = 结构化记忆 + 动态记忆

**Mem0 = 动态记忆层（向量数据库）**
- 自动学习用户偏好、模式
- 语义搜索，模糊匹配
- 对话式记忆，学习用户习惯

**Self-Improving = 自我反思系统（文件存储）**
- 记录用户纠正
- 自我反思日志
- 模式识别和提升

## 📝 Mem0 核心功能

### 1. 语义搜索

```bash
# 搜索用户偏好
node scripts/mem0-search.js "user preferences" --limit=3

# 搜索特定用户
node scripts/mem0-search.js "query" --user=username --limit=5
```

**技术原理：**
- 向量嵌入（Embedding）
- 余弦相似度
- Milvus/ChromaDB 存储

### 2. 记忆存储

```bash
# 显式存储
node scripts/mem0-add.js "用户喜欢简洁的更新"

# 对话存储（自动提取）
node scripts/mem0-add.js --messages='[{"role":"user","content":"..."}]'
```

**智能提取：**
- LLM 自动去重
- 合并相似记忆
- 提取关键偏好

### 3. 性能指标

| 指标 | 数据 |
|------|------|
| 准确率 | +26% over OpenAI Memory |
| 速度 | 91% faster |
| Token | 90% fewer |
| 延迟 | Sub-50ms |

## 📝 Self-Improving 核心功能

### 1. 三层存储架构

```
HOT (memory.md)     ≤100行  常驻内存
    ↓
WARM (projects/domains/)  ≤200行  按需加载
    ↓
COLD (archive/)     无限    归档存储
```

### 2. 学习信号

**✅ 应该记录：**
- 用户纠正："不，应该是..."
- 偏好信号："我喜欢..."
- 模式候选：重复3次的操作

**❌ 不要记录：**
- 一次性指令
- 特定上下文
- 假设性内容

### 3. 自我反思流程

```
完成工作 → 暂停评估 → 发现问题 → 记录教训
    ↓
教训重复3次 → 提升到 HOT 层
```

**反思格式：**
```
CONTEXT: [任务类型]
REFLECTION: [发现的问题]
LESSON: [下次改进]
```

### 4. 自动晋升/降级

| 条件 | 动作 |
|------|------|
| 7天内使用3次 | 提升到 HOT |
| 30天未使用 | 降级到 WARM |
| 90天未使用 | 归档到 COLD |

## ⚡ 怎么使用

### Mem0 工作流

```bash
# 1. 回答前搜索
node scripts/mem0-search.js "用户的项目偏好"

# 2. 对话后存储
node scripts/mem0-add.js --messages='[
  {"role": "user", "content": "我喜欢简洁的回复"},
  {"role": "assistant", "content": "好的，我会简洁"}
]'

# 3. 列出所有记忆
node scripts/mem0-list.js --user=username

# 4. 删除特定记忆
node scripts/mem0-delete.js <memory_id>
```

### Self-Improving 工作流

```bash
# 1. 记录纠正
# 当用户纠正时 → 添加到 corrections.md

# 2. 自我反思
# 完成重要工作后 → 评估结果 → 记录反思

# 3. 模式识别
# 发现重复模式 → 询问确认 → 升级为规则

# 4. 内存统计
# 定期检查 → 压缩 → 归档
```

### 组合使用

```bash
# 1. 回答前：搜索双重记忆
node scripts/mem0-search.js "用户偏好"      # Mem0
grep "pattern" ~/self-improving/memory.md  # Self-Improving

# 2. 回答后：存储新记忆
node scripts/mem0-add.js "新学习的内容"    # Mem0
# 记录反思到 corrections.md               # Self-Improving

# 3. 定期：整理记忆
# 压缩 memory.md
# 归档过期 patterns
```

## ✅ 优点

### Mem0

1. **语义搜索** - 自然语言查询，不需要精确关键词
2. **自动学习** - 不需要手动维护
3. **高性能** - 毫秒级响应
4. **去重合并** - LLM 自动整理
5. **跨会话** - 记住所有历史对话

### Self-Improving

1. **自我进化** - 从错误中学习
2. **分级存储** - 热/温/冷分层，节省资源
3. **模式识别** - 发现重复规律
4. **透明性** - 记录可追溯
5. **自动化** - 晋升/降级自动执行

---

## ❌ 缺点

### Mem0

1. **依赖外部服务** - 需要 OpenAI API
2. **向量数据库** - 需要 Milvus/ChromaDB
3. **延迟** - 比本地文件慢
4. **成本** - API 调用有费用

### Self-Improving

1. **文件管理复杂** - 多层目录结构
2. **手动整理** - 需要定期维护
3. **容量限制** - HOT 层 ≤100 行
4. **学习成本** - 需要理解分层逻辑
