# Karpathy 知识库工作流

## 核心理念

Karpathy 知识库工作流是一种利用大型语言模型（LLM）将原始资料编译成结构化的知识库的方法。这种方法通过三个步骤实现：收集、编译和查询。

## 工作流三步曲

### STEP 01: 收集
- **raw/** 目录存放原始资料，包括但不限于论文、文章、GitHub 代码、数据集或截图。
- 使用 Obsidian Web Clipper 插件保存网页内容到 raw/ 文件夹中。

### STEP 02: 编译
- LLM 读取 raw/ 下的所有文件，并提取核心概念。
- 自动生成结构化的 wiki 文章，同时建立双向链接关系，确保知识库内部的逻辑连贯性与信息检索效率。

### STEP 03: 查询
- 可直接向 LLM 提问，获取所需信息。
- LLM 将在已构建的知识库（wiki/）中查找答案，并提供相关文档或内容作为参考依据。对于好的回答，可以将其归档回知识库以供后续使用。

## 为什么不需要 RAG？

当处理规模为 100 篇文献、总计约 40 万字的资料时，LLM 的上下文窗口已经足够大，可以直接将整个 wiki 内容纳入其 context 中进行查询和生成。因此，在这种情况下，并不必要采用 Retrieval-Augmented Generation (RAG) 技术。

## 相关概念

- [[LLM]]：大型语言模型
- [[Obsidian]]：一款支持 Markdown 编写的笔记软件，内置插件 Obsidian Web Clipper 可以帮助收集网页内容。
- [[知识管理]]：一种系统化的方法来创建、组织和分享信息资源，旨在提高个人或团队的工作效率与创新能力。
- [[RAG]]：检索增强生成技术，在需要时从外部数据源中检索信息并结合模型生成结果。
