## 核心原理

### 三层架构

| 层级 | 目录/文件 | 维护者 |
|------|-----------|--------|
| Raw Sources（原始资料） | `raw/` | 👤 只有用户可添加 |
| Wiki（知识网络） | `wiki/` | 🤖 AI 全权维护 |
| Schema（规则手册） | `CLAUDE.md` | 👤 + 🤖 共同维护 |

### 三种核心操作（闭环）

1. **INGEST 摄入**: AI 自动阅读 raw/ 资料 → 提炼 → 更新 Wiki（一次摄入影响10-15个页面）
2. **QUERY 查询**: 先读 index.md → 深读相关页面 → 综合回答（好答案可沉淀回 Wiki）
3. **LINT 维护**: 健康检查 → 矛盾检测 → 孤儿页面 → 过时信息

### 人机分工

**人类负责**: 筛选员 + 导航员 + 提问者 + 决策者
**AI 负责**: 阅读员 + 链接员 + 审计员 + 归档员

**核心理念**: "把枯燥的书记员工作交给 AI，你只负责提供资料和提好问题"

## 目录结构

```
knowledge-base/
├── CLAUDE.md        # Schema - AI的说明书
├── raw/             # 原始资料（只读）
│   └── [日期/主题]/  # 具体资料
├── wiki/            # 知识库（AI维护）
│   ├── index.md     # 内容索引（每次ingest后自动更新）
│   ├── log.md      # 操作日志（仅追加）
│   ├── 实体/       # 人/公司/产品/工具
│   ├── 概念/       # 原理/术语/方法论
│   ├── 来源/       # 原始资料摘要
│   └── 比较/       # 跨资料对比分析
└── [其他文件]
```

## Lint 健康检查清单

| 检查项 | 说明 |
|--------|------|
| 互斥矛盾 | 不同页面对同一事实描述不一致 |
| 孤儿页面 | 有链接但目标页面不存在 |
| 缺失实体/概念 | 有链接但目标不存在 |
| 过时信息 | 被新资料推翻但未更新 |
| index.md 同步 | 索引是否与实际文件同步 |

## 三路检索 vs RAG

| 特点 | RAG | LLM Wiki（知识库） |
|------|-----|-------------------|
| 方式 | 每次重新检索原始文档片段 | 增量式构建 + 持久化 + 结构化 |
| 累积 | 无累积，每次从零开始 | 持久化，相互链接 |
| 答案质量 | 依赖检索质量 | 知识越丰富，答案越精准 |

## 适合的场景

✅ **适合**: 专题研究 + 高质量资料的系统化整理 + 领域学习
❌ **不适合**: 个人笔记（内容碎片化、频繁修改）

## 配套工具

- **Obsidian Web Clipper**: Chrome 官方插件，一键保存网页/YouTube为 Markdown
- **compile.py**: ingest 脚本，raw → wiki 自动编译
- **query.py**: 查询脚本，从 Wiki 综合回答
- **lint.py**: 健康检查脚本
- **sync_all.py**: 全量同步（编译 + 飞书 + Notion + Milvus）
