# Karpathy 知识库工作流

## 核心理念

用[[LLM]]（大型语言模型）把原始资料（论文/文章/代码）编译成wiki知识库。

## 工作流三步曲

### STEP 01: 收集

- raw/目录存放原始资料。
- 支持：论文、文章、GitHub、数据集、截图。
- 使用[[Obsidian Web Clipper]]保存网页。

#### 实例
假设你正在阅读一篇关于机器学习的文章，你可以使用 Obsidian Web Clipper 将该页面保存到 raw/ 目录中。这样，原始资料就会被安全地存储在知识库中。

### STEP 02: 编译

- LLM读取raw/所有文件。
- 提取出核心概念。
- 生成结构化wiki文章。
- 建立双向链接。

#### 实例
假设你已经收集了一篇关于自然语言处理的文章，LLM 可以从该文章中提取关键信息，并将其编译成一个结构化的 wiki 页面。同时，这个页面会与相关概念建立双向链接，方便后续查询和理解。

### STEP 03: 查询

- 直接向LLM提问。
- LLM在wiki/中查找答案。
- 好答案归档回wiki。

#### 实例
当你对某个概念或问题感到困惑时，可以直接向 LLM 提问。例如，“什么是自然语言处理？”LLM 会从 wiki 中找到相关页面并提供详细的解释。好的回答会被归档回 wiki，以供未来参考。

## 为什么不需要RAG？

100篇/40万字规模下，LLM的上下文窗口足够大，直接把所有wiki内容放进context就够了。

#### 相关概念
- [[RAG]]（检索增强生成）：一种结合了检索和生成的方法，在特定场景下可能比单纯使用 LLM 更有效。但在 Karpathy 的知识库工作流中，由于 LLM 上下文窗口足够大，直接将所有内容纳入上下文即可满足需求。
