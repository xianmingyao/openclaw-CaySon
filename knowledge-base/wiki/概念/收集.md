# 收集

## 概念简介

收集是 Karpathy 工作流中的第一步，用于整理和保存原始资料（如论文、文章、代码等），为后续的编译和查询做准备。

## 详细步骤

### STEP 01: 收集

- **存放位置**：使用 `raw/` 目录来存储所有原始资料。
- **支持类型**：
  - 论文
  - 文章
  - GitHub 代码
  - 数据集
  - 截图
- **工具应用**：通过 Obsidian Web Clipper 插件保存网页内容，方便快捷地收集互联网上的信息。

## 实例

假设你正在研究一个新发布的机器学习论文。你可以将该论文的 PDF 文件、相关 GitHub 代码链接以及一些关键截图都存放在 `raw/` 目录中，并使用 Obsidian Web Clipper 快速保存论文中的重要网页片段，确保所有资料齐全且易于访问。

## 与其他步骤的关系

- **与编译（STEP 02）**：收集阶段完成后，LLM 将读取 `raw/` 目录下的所有文件进行处理。[[编译]]过程将提取核心概念并生成结构化的 wiki 文章。
- **与查询（STEP 03）**：通过直接向 LLM 提问，可以快速获取所需信息。如果 LLM 在 wiki 中找到了答案，则这些好答案会被归档回 wiki，以供后续参考。

## 相关概念

[[LLM]]：大型语言模型，用于处理和生成文本。
[[Obsidian]]：一个强大的知识管理工具，支持笔记、链接管理和插件扩展。
[[知识管理]]：一种系统化的方法来组织、存储和检索信息。
[[RAG]]（Retrieval-Augmented Generation）：一种结合检索与生成技术的模型，用于增强 LLM 的查询能力。

通过上述步骤和概念的理解，你可以高效地构建一个结构化的知识库，并利用 LLM 实现高效的查询和学习。
