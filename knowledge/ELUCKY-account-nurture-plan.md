## 一、平台风控机制深度解析

### 1.0 TikTok 风控机制（2026年最新）

#### 核心风控维度

| 风控维度 | 具体内容 | 触发阈值 |
|---------|---------|---------|
| **IP稳定性** | IP类型（原生/机房）、IP归属地 | 机房IP可接受，原生IP更佳 |
| **设备指纹** | 模拟器检测、浏览器指纹 | 检测到模拟器直接封 |
| **行为模式** | 操作频率、时间间隔 | 明显机器人特征直接封 |
| **账号历史** | 注册时间、资料完善度 | 新号快速操作直接封 |
| **内容特征** | 原创度、敏感词、搬运检测 | 搬运内容易被检测 |
| **连坐风险** | 低（与其他平台独立） | 风险相对较低 |

#### TikTok 限流规则

| 操作类型 | 每日限制 | 新账号限制 |
|---------|---------|----------|
| 关注/取关 | 200-500人/天 | 50-100人/天 |
| 点赞 | 500-1000次/天 | 100-200次/天 |
| 评论 | 200-300次/天 | 30-50次/天 |
| 发视频 | 3-5次/天 | 1-2次/天 |
| 私信 | 50-100条/天 | 10-20条/天 |
| 分享 | 100-200次/天 | 20-30次/天 |

#### TikTok 封号触发场景

```
🔴 高危场景（直接封号）
├── 使用模拟器或第三方工具（检测到即封）
├── 新号快速大量操作
├── 大量发布搬运/抄袭内容
├── 频繁切换IP（跨地域登录）
├── 发布违规内容（政治、色情、敏感词）
└── 被多人举报

🟡 中危场景（功能限制）
├── 频繁操作超出限制
├── 内容原创度低（被判搬运）
├── 短时间内大量点赞/关注
├── 使用非原生IP（机房IP）
└── 异常登录时间

🟢 低危场景（警告）
├── 频繁修改资料
├── 使用第三方滤镜/编辑软件
├── 视频质量不稳定
└── 粉丝增长速度异常
```

#### TikTok 健康评分指标

| 健康指标 | 权重 | 说明 |
|---------|------|------|
| **播放量** | 原有体系 | 反映内容质量 |
| **限流次数** | 高权重 | 触发次数越多越危险 |
| **验证码触发** | 中权重 | 频繁触发说明行为异常 |
| **Shadow Ban** | 高权重 | 被限流但不自知 |

#### TikTok Shadow Ban 特征

- **检测方法**：发布视频后用另一个账号搜索自己的用户名，如搜不到=被Shadow Ban
- **持续时间**：通常7-30天
- **解除方法**：停止所有操作7天+发布原创内容

## 二、养号安排方案

### 2.0 TikTok 养号方案（核心重点）

#### 新号养号周期（14天）

| 阶段 | 天数 | 核心任务 | 每日操作上限 |
|------|------|---------|-------------|
| **初期适应** | Day 1-3 | 完善资料、养浏览习惯 | 点赞20次、关注10人、评论5条 |
| **社交建立** | Day 4-7 | 关注目标账号、互动 | 点赞50次、关注30人、评论10条 |
| **信任积累** | Day 8-10 | 发布首条视频、互动 | 点赞100次、关注50人、评论20条 |
| **稳定期** | Day 11-14 | 保持活跃、发布频率 | 正常上限的50% |
| **成熟期** | Day 15+ | 正式运营 | 正常上限 |

#### TikTok 养号操作规范

```markdown
✅ 正确操作
├── 使用真实头像和资料
├── 先浏览30分钟再进行互动（模拟真人）
├── 关注垂直领域相关账号
├── 互动时添加真实评论
├── 发布高质量原创视频（9:16比例，1080x1920）
├── 使用Hashtag（3-5个/视频）
├── 保持账号持续活跃
└── 固定IP登录（原生IP更佳）

❌ 禁止操作
├── 使用模拟器或第三方工具
├── 新号快速大量关注/点赞
├── 发布搬运/抄袭内容
├── 频繁切换IP
├── 使用机房IP（风险较高）
├── 大量发布外链
└── 使用敏感词或违规内容
```

#### TikTok 视频规格要求

| 内容类型 | 比例 | 分辨率 | 时长 |
|---------|------|--------|------|
| 视频 | 9:16 | 1080x1920 | 15秒-10分钟 |
| 封面 | 9:16 | 1080x1920 | - |

### 3.0 TikTok 挂靠方案（核心重点）

#### 根基账号标准

| 标准 | 要求 |
|------|------|
| 账号年龄 | >6个月 |
| 粉丝数量 | >10,000 |
| 活跃度 | 近30天有发布 |
| 违规记录 | 无封号/限流历史 |
| 内容质量 | 高（原创、垂直） |

#### TikTok 挂靠策略

```
根基账号（大号）
├── 开通Creator Fund（证明是真实创作者）
├── 关注小号，建立关注链
├── 大号点赞/评论小号内容（提升曝光）
├── 直播间互动（提升活跃度）
└── 内容联动（duet/stitch）

新号/小号
├── 关注根基账号
├── 根基账号内容必点赞评论
├── 发布内容@根基账号
└── 学习根基账号的内容风格
```

#### TikTok 引流模式

```
关注 -> 私信 -> WA（WhatsApp）
日均WA新增：60-90
日均注册：36-63
```

### 4.4 风控应急预案

#### 一级警报：功能限制

```
触发条件：被限制评论/私信/发布功能
应对措施：
├── 立即停止所有操作
├── 保持账号正常浏览
├── 24小时内不进行任何互动
├── 24小时后逐步恢复（50%操作量）
├── 一周内逐步恢复至正常
└── 记录触发原因，避免再次触发
```

#### 二级警报：账号锁定

```
触发条件：账号被锁定，需要验证
应对措施：
├── 通过邮件/手机验证解锁
├── 解锁后保持静默48小时
├── 修改密码但保持IP不变
├── 一周内保持低频操作
└── 检查是否有违规内容
```

#### 三级警报：封号

```
触发条件：账号被封禁
应对措施：
├── 立即停止该账号所有操作
├── 检查关联账号是否有风险
├── 通过正规渠道申诉（根基账号）
├── 内容号/引流号直接启用备用号
├── 排查封号原因（IP/设备/行为）
└── 更新风控规则，避免再次发生
```

## 五、跨平台连坐风控（Meta特别重要）

### 5.1 Facebook + Instagram 连坐规则

**同一设备关联**
- FB+IG同设备登录 → 一个封另一个受影响
- **隔离方案**：FB和IG号分开Profile

**同一IP关联**
- 同一IP下的多个账号被标记协同
- **隔离方案**：每10个号用不同IP段

**手机号关联**
- 同一手机号绑定的FB+IG一起封
- **隔离方案**：每个号独立手机号

**邮箱关联**
- 同一邮箱域名批量注册被检测
- **隔离方案**：使用不同邮箱服务

**行为模式关联**
- 多个号同时操作/内容相似
- **隔离方案**：错开操作时间/内容差异化
