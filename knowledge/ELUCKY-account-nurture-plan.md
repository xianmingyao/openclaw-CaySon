## 一、平台风控机制深度解析

### 1.1 Facebook 风控机制

#### 核心风控维度

| 风控维度 | 具体内容 | 触发阈值 |
|---------|---------|---------|
| **IP稳定性** | IP归属地、IP段一致性 | 同一IP登录超过5个账号 |
| **设备指纹** | 浏览器指纹、Canvas、WebGL、UA | 同一设备登录超过3个账号 |
| **行为模式** | 操作频率、时间间隔 | 明显机器人特征 |
| **账号历史** | 注册时间、初始信息完善度 | 新号快速操作 |
| **社交关系** | 好友数量、互动频率 | 批量添加好友 |
| **内容特征** | 链接比例、敏感词、模板化 | 外链超过30%、诱导转发 |

#### Facebook 限流规则

| 操作类型 | 每日限制 | 新账号限制 |
|---------|---------|----------|
| 加好友 | 20-50人/天 | 5-10人/天 |
| 点赞 | 200-500次/天 | 50-100次/天 |
| 评论 | 100-200次/天 | 20-50次/天 |
| 发帖 | 5-10次/天 | 2-3次/天 |
| 私信 | 15-30条/天 | 3-5条/天 |
| 加入群组 | 10-20个/天 | 3-5个/天 |

#### Facebook 封号触发场景

```
🔴 高危场景（直接封号）
├── IP突然跨地域（香港→美国→巴西）
├── 新号大量添加好友
├── 频繁切换设备
├── 发布违规内容
└── 被多人举报

🟡 中危场景（功能限制）
├── 频繁操作超出限制
├── 短时间内大量点赞
├── 模板化内容
└── 异常登录时间

🟢 低危场景（警告）
├── 频繁切换浏览器
├── 短时间内多次修改资料
└── 使用第三方工具
```

## 二、养号安排方案

### 2.1 养号核心理念

**养号 = 与平台建立信任**

不是刷数据，而是让账号表现得像一个"真实用户"。

### 2.2 Facebook 养号方案

#### 新号养号周期（14天）

| 阶段 | 天数 | 核心任务 | 每日操作上限 |
|------|------|---------|-------------|
| **初期适应** | Day 1-3 | 完善资料、浏览内容 | 点赞5次、关注3人、评论2条 |
| **社交建立** | Day 4-7 | 添加少量好友、加入群组 | 点赞20次、关注10人、评论10条 |
| **信任积累** | Day 8-10 | 偶尔发帖、互动 | 点赞50次、关注20人、评论20条 |
| **稳定期** | Day 11-14 | 正常运营 | 正常上限的50% |
| **成熟期** | Day 15+ | 正式运营 | 正常上限 |

#### Facebook 养号操作规范

```markdown
✅ 正确操作
├── 每天固定时间操作，模拟真人作息
├── 先浏览10-15分钟再进行互动
├── 互动时添加真实评论（非模板）
├── 完善个人资料（头像、封面、简介）
├── 绑定手机号和邮箱
├── 经常保持登录状态
└── 发帖频率：每2-3天1篇

❌ 禁止操作
├── 新号大量添加好友（>20人/天）
├── 注册后立即大量操作
├── 使用VPN频繁切换IP
├── 发布含外链的营销内容
├── 重复发送相同内容
└── 使用第三方自动化工具
```

## 三、挂靠安排方案

### 3.1 挂靠概念

**挂靠 = 将账号绑定在稳定的"根基账号"下，通过根基账号的信任度带动新号/小号**

### 3.2 Facebook 挂靠方案

#### 根基账号标准

| 标准 | 要求 |
|------|------|
| 账号年龄 | >1年 |
| 好友数量 | >500人 |
| 活跃度 | 近30天有互动 |
| 违规记录 | 无封号历史 |
| 账号权重 | 高（经常互动、被互动） |

#### 挂靠策略

```
根基账号（大号）
├── 开设专页（Page），新号关注专页
├── 管理员添加新号为管理员
├── 大号点赞/评论新号内容
├── 群组关联（同一群组）
└── 互动关联（经常互相点赞评论）

新号/小号
├── 关注根基账号的专页
├── 加入根基账号管理的群组
├── 与根基账号保持互动
└── 资料信息与大号有差异
```

#### 账号矩阵挂靠关系

```
FB账号矩阵挂靠图

大号A（根基）
├── 内容号A1（挂靠）
├── 内容号A2（挂靠）
├── 引流号A3（挂靠）
└── 备用号A4（挂靠）

大号B（根基）
├── 内容号B1（挂靠）
├── 内容号B2（挂靠）
└── ...
```

### 4.4 风控应急预案

#### 一级警报：功能限制

```
触发条件：被限制评论/私信/发布功能
应对措施：
├── 立即停止所有操作
├── 保持账号正常浏览
├── 24小时内不进行任何互动
├── 24小时后逐步恢复（50%操作量）
├── 一周内逐步恢复至正常
└── 记录触发原因，避免再次触发
```

#### 二级警报：账号锁定

```
触发条件：账号被锁定，需要验证
应对措施：
├── 通过邮件/手机验证解锁
├── 解锁后保持静默48小时
├── 修改密码但保持IP不变
├── 一周内保持低频操作
└── 检查是否有违规内容
```

#### 三级警报：封号

```
触发条件：账号被封禁
应对措施：
├── 立即停止该账号所有操作
├── 检查关联账号是否有风险
├── 通过正规渠道申诉（根基账号）
├── 内容号/引流号直接启用备用号
├── 排查封号原因（IP/设备/行为）
└── 更新风控规则，避免再次发生
```
