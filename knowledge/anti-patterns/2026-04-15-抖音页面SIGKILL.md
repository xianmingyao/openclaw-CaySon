# 抖音页面SIGKILL问题

## 问题现象
通过 agent-browser 打开抖音视频页面时，进程被系统SIGKILL杀死

## 根本原因
- 抖音重定向链接（v.douyin.com/xxx）会被系统拦截
- 页面加载时触发安全机制被kill

## 解决方案
**方案1：直接用视频ID访问（推荐）**
```bash
# 错误：会被SIGKILL
npx agent-browser open "https://v.douyin.com/sFEAvt0trms/"

# 正确：先用短链解析得到真实URL，再访问
# 例如解析得到 https://www.douyin.com/video/7627021764320940282
npx agent-browser open "https://www.douyin.com/video/7627021764320940282"
```

**方案2：等待重定向完成后再操作**
1. 打开短链
2. 等待重定向
3. 获取真实URL再继续

## 如何避免
1. **优先使用视频ID**：有ID就不用短链
2. **提前解析短链**：用其他方式解析短链得到真实URL
3. **SIGKILL后重试**：偶尔能成功，可能是竞态条件
