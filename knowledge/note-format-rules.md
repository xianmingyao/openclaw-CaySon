# 宁兄笔记格式规范（铁律）

> 来源：2026-03-28 宁兄提供

## 核心格式原则

**必须遵守的格式要素：**

### 1. 标题层级
- ❌ 不用：传统H1/H2多级嵌套
- ✅ 应用：扁平化模块设计，用emoji和颜色区分重点

### 2. Emoji图标系统
```
✅ 正面反馈（好消息、成功状态）
🔴 问题陈述（报错、错误）
💡 提示说明
📝 步骤指引
🎯 核心要点
```

### 3. 标题格式
- 一级强调：✅/🔴 + emoji + 加粗文字
- 二级引导：加粗文字（如"第一步：xxx"）
- 总结部分：用"总结"作为收尾标题

### 4. 代码块格式（必须严格遵守）
```
┌─ plaintext ──────────────────────────────────┐
│ [gateway] listening on ws://127.0.0.1:18789  │
└─────────────────────────────────────────────┘
```
- 外框：浅灰色圆角边框
- 背景：极浅灰色
- 顶部栏：标注语言类型（bash/plaintext/python）
- 字体：等宽字体(Monospace)
- 语法高亮：命令深色，路径字符串用绿色/棕色

### 5. 列表格式
- 有序列表（数字）：用于具体错误点、步骤
- 无序列表（实心圆点）：用于列举效果、要点
- 每行内容简洁，方便快速扫描

### 6. 分割线与区块
- 浅灰色细横线分隔不同逻辑块
- 白色背景 + 代码块灰色背景 = 视觉层次

### 7. 排版规范
- 左对齐
- 行间距适中
- 段落间有明显垂直空隙
- 留白充足，避免视觉压迫

### 8. 交互式视觉元素
- ✅ 绿色方块勾选 = 任务完成/成功
- 🔴 橙色/红色 = 存在问题/警示

## 示例模板

```markdown
✅ 重大好消息
核心已正常运行

🔴 现在的报错只是两个小问题
1. 飞书组件版本不兼容
2. Discord组件网络超时

┌─ bash ──────────────────────────────────────┐
│ rd /s /q "C:\Users\...\feishu"            │
└─────────────────────────────────────────────┘

第一步：删除坏掉的飞书插件
• 使用 rd /s /q 命令强制删除

总结（超级简单）
```

## 格式检查清单

- [ ] 是否有emoji图标区分正负面？
- [ ] 标题是否加粗？
- [ ] 代码块是否有语言标签？
- [ ] 代码块是否用等宽字体？
- [ ] 列表是否清晰简洁？
- [ ] 区块间是否有分割线？
- [ ] 整体是否左对齐、留白充足？

**铁律：格式不对必须修改，绝不妥协！**
