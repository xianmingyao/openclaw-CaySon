# PRD Writer Skill 研究报告

## 📌 一句话定位

> **把产品经理的沟通经验和产品思维写进Skill，让AI半小时写出直接能用的PRD**

## 🎯 核心痛点解决

| 痛点 | 解决方案 |
|------|----------|
| AI写的PRD太长太专业但不懂需求 | 先概念版对齐方向，再落地版补充细节 |
| AI忽略交互细节（报错/等待/重复点击） | 五大盲区补全清单 |
| PRD没法直接给开发用 | 三视角诊断（用户/商业/技术）|

## 📋 工作流程（5步）

```
阶段1: 需求采集 → 阶段2: Feature List → 阶段3: PRD文档
    → 阶段4: 原型生成 → 阶段5: Cloudflare部署
```

### 概念版 vs 落地版

| 阶段 | 时间 | 重点 |
|------|------|------|
| **概念版** | 10min | 三视角诊断，对齐方向 |
| **落地版** | 30min | 五大盲区补全 |

### 三视角诊断

| 视角 | 问题 |
|------|------|
| 用户视角 | 痛点在哪？真帮到了吗？ |
| 商业视角 | 一次性还是长期？买断还是订阅？ |
| 技术视角 | 产品形态？能快速做出来？ |

### 五大盲区补全

1. 交互细节（报错/等待/重复点击）
2. 状态机（所有状态+转换）
3. 字段规范（必填/长度/校验）
4. 文案规范（用户端+开发端）
5. 关键图纸（流程图+线框图）

## 🛠️ 技术亮点

### UI-UX-Pro-Max 设计系统
- 内置设计数据CSV（配色/字体/图标/技术栈）
- 支持生成完整设计系统
- 覆盖10+技术栈（React/Next.js/Vue/Svelte/SwiftUI/Flutter等）

### 原型生成
- 生成单文件HTML（Tailwind CSS + 原生JS）
- 可交互原型
- Cloudflare Pages一键部署

### 质量检查（四角色审视）
1. 技术负责人：实现难度/性能/安全
2. 挑剔用户：操作便捷性/流程合理性
3. 运营负责人：数据分析/营销推广
4. 测试工程师：异常场景/边界问题
