### 5. 技术教训

**PowerShell stderr 问题**：
- PowerShell 把所有 stderr 输出当作 "native command error"，即使只是日志
- 解决方案：用 `subprocess.run()` 包装 action 执行，在子进程中 `sys.stderr = open(os.devnull, 'w')`
- 另一个方案：设置 `os.environ["LOGURU_LEVEL"] = "ERROR"` 抑制日志

**ctypes WINFUNCTYPE callback**：
- 回调函数中不能定义 ctypes Structure 类（需要先定义再实例化）
- 必须用 `try/except` 包裹回调内的所有代码，否则异常会导致 EnumWindows 崩溃
- callback 闭包中引用的变量需要是线程安全的

**jingmai-agent bug**：
- `ui_collect_actions.py` 中 `_enum_windows_proc` 的 `results` 是 `LP_c_long` 类型（ctypes 指针），不是 Python list
- `CaptureWindowScreenshotAction` 导入了不存在的 `crop_to_active_window`
- 都需要在 CLI 层做 workaround

### 📌 主题一：OpenClaw记忆持久化方案（"小龙虾"不遗忘）

**痛点问题**：
- OpenClaw每次新建会话就完全失忆
- 明明刚教会完整流程，切换会话后AI就忘了
- 这不是模型缺陷，是没有正确使用三层独立记忆体系

**核心解决方案**：
1. **mem0 + COS Vectors 方案**
   - 插件名：openclaw-mem0
   - 原理：将记忆存储在上下文窗口外部
   - Auto-Recall机制：响应前自动召回相关记忆
   - Auto-Capture机制：响应后自动捕获新信息

2. **三层独立记忆体系**
   - 第一层：上下文窗口（当前会话）
   - 第二层：自动召回（向量检索）
   - 第三层：长期存储（文件/数据库）

**使用场景**：跨会话记住用户偏好、持续学习的工作流、知识积累型任务

**安装配置**：
`ash
# 安装 mem0 插件
openclaw skills install mem0
# 配置 COS Vectors（腾讯云）或本地部署 + 百炼API
`
