# AI 产品策略模式参考

常见 AI 产品策略组件的标准描述框架。

---

## 1. 推荐系统策略

**适用场景：** 内容推荐、商品推荐、用户匹配

### 标准三层架构
```
召回层 → 粗排层 → 精排层 → 重排/业务规则
```

**PRD 描述要点：**
- 召回源：协同过滤 / 内容相似 / 热门 / 关注关系
- 排序目标：CTR / 时长 / 转化 / 多目标加权
- 业务约束：去重、打散、品类均衡、广告插入位
- 冷启动：新用户用人口属性/热门兜底，新内容用内容质量分

---

## 2. 搜索策略

**适用场景：** 站内搜索、语义检索

**PRD 描述要点：**
- 查询理解：分词、纠错、意图识别、Query 改写
- 召回：倒排索引 / 向量检索 / 混合召回
- 排序：相关性 × 质量分 × 个性化
- 兜底：无结果时的降级展示逻辑

---

## 3. 意图识别策略

**适用场景：** 对话系统、智能客服、NLU

**PRD 描述要点：**
- 意图分类体系（树形结构）
- 置信度阈值设计（高置信直接执行 / 低置信澄清确认）
- 多意图处理
- 兜底：未识别意图的转人工/通用回复逻辑

---

## 4. 内容审核策略

**适用场景：** UGC 平台内容安全

**PRD 描述要点：**
- 审核维度：违规类型体系
- 处置策略：机审 → 人审 → 申诉
- 分级处置：不展示 / 限流 / 警告 / 封号
- 误伤率控制目标

---

## 5. 个性化策略

**适用场景：** 首页、push、邮件个性化

**PRD 描述要点：**
- 用户画像：兴趣标签 / 行为序列 / 人口属性
- 个性化程度：全个性化 / 混合（个性化+热门）/ 纯热门
- 隐私合规：数据使用说明，用户可退出个性化

---

## 6. 定价/激励策略

**适用场景：** 动态定价、优惠券发放、积分体系

**PRD 描述要点：**
- 触发条件：用户行为 / 时间窗口 / 库存状态
- 激励力度：金额范围、发放规则
- 防滥用：频率限制、用户分层
- 效果衡量：增量 ROI（而非绝对 GMV）

---

## 通用策略 PRD Checklist

- [ ] 明确策略优化目标（单一核心指标）
- [ ] 列出输入信号及数据来源
- [ ] 描述策略逻辑（伪代码或流程图）
- [ ] 定义兜底规则（策略不可用时的降级）
- [ ] 设计 A/B 实验方案
- [ ] 说明冷启动处理
- [ ] 列出数据/隐私合规要求
