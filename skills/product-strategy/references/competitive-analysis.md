# Competitive Analysis

Real competitive analysis answers a decision, not a curiosity. Start with the question: enter, reposition, price, defend, or partner. If you do not know the decision, the output turns into a feature spreadsheet with no consequence.

## Jobs To Be Done

Define the job the product is hired for before comparing vendors.

- Functional job: what task the customer needs done.
- Emotional job: how the customer wants to feel while doing it.
- Social job: what identity or status the purchase signals.

Compare competitors by how they solve the same job, not by whether they share a category label. A portable speaker and a smart display may compete for the same kitchen audio job. So what: the true competitor set is broader than the shelf set.

## Feature Parity Matrix

Use a matrix only after the job is clear.

- Rows: capabilities or buying criteria that materially affect choice.
- Columns: products or companies in the relevant segment.
- Include: table stakes, switching triggers, monetization hooks, and trust factors.
- Exclude: vanity features, niche edge cases, and items no buyer uses in evaluation.

Structure each feature row with four fields:

1. Customer importance.
2. Current winner.
3. Ease of replication.
4. Strategic implication.

If a feature can be copied in one product cycle, it is not a moat. So what: prioritize asymmetries that are hard to reproduce, not headline parity.

## Positioning Map Construction

Positioning maps work only when the axes explain purchase behavior.

- Choose axes customers actually use to trade off value.
- Prefer one functional axis and one commercial or experience axis.
- Avoid axes where everyone clusters by definition, like "quality" versus "innovation."
- Build separate maps for different segments if budget and premium buyers use different criteria.

After plotting:

- Identify clusters.
- Note empty spaces.
- Test whether empty spaces are attractive or simply unattractive demand.
- Mark which players have permission to move across clusters.

So what: whitespace is only useful if customers want it and the channel can support it.

## Competitive Moats

Classify moats explicitly:

- Network effects: value rises as more users participate.
- Switching costs: data, workflow, accessories, habits, or retraining make exit painful.
- Scale economies: cost advantages in sourcing, distribution, or support.
- Brand: trusted shortcut for quality, identity, or status.
- IP: patents, protected tech, regulated approvals, or exclusive rights.

Ask three questions for every moat claim:

1. Is it real today?
2. How long does it last?
3. Does it matter to the buying decision?

So what: many products have strengths, but few have defensibility.

## The "So What?" Test

Every observation must end with an implication:

- Observation: Competitor A bundles subscription support.
- So what: churn falls, switching costs rise, and price comparison becomes harder.

If an observation cannot change a decision, cut it.

## Anti-Patterns

- Feature lists without insight.
- Maps with arbitrary axes chosen because they look clean.
- "We are better at everything" conclusions.
- Treating every player as equally relevant.
- Confusing marketing claims with sustained advantage.

The output should make a decision easier: where to attack, where not to play, and what would need to be true to win.
