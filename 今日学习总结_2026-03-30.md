## 技术架构/方法论

### 1. 多智能体协作架构（Multi-Agent Collaboration）

**核心模式：**
- **主智能体 + 子智能体**：主智能体负责任务分解和结果汇总
- **并行执行**：多个子智能体同时处理不同子任务
- **交接机制（Swarm）**：不同Agent像高效团队一样协同工作

**关键框架：**
- LangGraph 1.0（字节deer-flow使用）
- Agent Teams（多独立上下文并行）
- Agent Swarm（OpenAI实验性框架）

**设计原则：**
- 每个Agent职责单一明确
- 通过消息传递而非共享状态通信
- 支持动态增减Agent角色

### 2. 记忆系统架构（Memory System Architecture）

**分层设计：**
```
长期记忆（永久保存）
├── MEMO.md - 核心身份和偏好
├── memory/ - 每日日志
└── knowledge/ - 知识库

短期记忆（会话级）
├── 会话上下文
└── 向量检索（Milvus/ChromaDB）
```

**评分机制：**
- ≥4分 → 长期记忆
- 2-3分 → 当日日志
- <2分 → 不记录

**技术选型：**
- 向量数据库：Milvus（云端）+ ChromaDB（本地）
- Embedding模型：nomic-embed-text
- 检索策略：优先Milvus，失败降级ChromaDB

### 3. VibeCoding 开发方法论

**核心思想：** AI辅助开发，强调快速原型和迭代式构建

**开发前9步：**
1. 明确需求和目标
2. 确定技术栈
3. 设计基本架构
4. 建立开发环境
5. 创建原型
6. 迭代开发
7. 集成测试
8. 优化性能
9. 部署上线

**开发中5个关键点：**
1. 持续沟通需求
2. 及时确认AI输出
3. 分块验证功能
4. 保持代码整洁
5. 及时重构优化

### 4. Harness Engineering（Agent可靠性工程）

**三大核心：**
- **上下文隔离**：防止信息混乱，每个任务独立上下文环境
- **计划执行验证**：逐步确认每步结果，异常及时回滚
- **熵的治理**：控制AI输出不确定性，减少幻觉

**关键技术：**
- 沙箱隔离（Docker）
- 计划图谱（任务分解和依赖管理）
- 验证点设置
- 回滚机制
